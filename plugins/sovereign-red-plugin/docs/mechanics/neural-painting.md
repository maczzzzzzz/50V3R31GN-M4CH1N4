## ⚡ THE NEURAL UPLINK (CDP_CORE)
The **Neural Uplink** provides the system with native "Physical Eyes" into the simulation. By tapping directly into the Foundry VTT GPU buffer via the **Chrome DevTools Protocol (CDP)**, the RED Plugin achieves hardware-level integration.

### ◈ CAPABILITIES
- **VISUAL_GROUNDING**: Capture of raw rendering buffers via the `CDP-Mesh`. The AI perceives the canvas with 1:1 fidelity.
- **GHOST_REFRESH**: Automated window re-ignition for module synchronization and CSS hot-loading.
- **SOVEREIGN_HIJACK**: Real-time injection of glitch-FX and narrative overlays directly into the Foundry renderer via `SOVEREIGN_HIJACK_JS`.

---

## 🎨 THE NEURAL PAINTER (BATCH_INGESTION)
The **Neural Painter** is the materialization vector for the **RED_DIRECTOR**. It bypasses standard WebSocket latency by executing batch creation commands directly within the renderer's execution context.

### ◈ CORE FUNCTIONS
- **ATOMIC_SPAWNING**: Materialize entire NPC squads and environmental tokens in a single execution cycle.
- **GEOMETRIC_FORGING**: Automated construction of walls, portals, and light sources derived from Computer Vision analysis of raw map data.
- **ATMOSPHERE_PULSE**: Real-time synchronization of illumination and darkness levels with the Nucleus Deck's sensory telemetry.

---

## 🛠️ INFRASTRUCTURE REQUIREMENTS
1. **BOOT_FLAGS**: Foundry VTT must be initiated with `--remote-debugging-port=9222`.
2. **AKASHIK_SYNC**: A valid `sceneId` must be shored within **Akashik.db** for targeting.
3. **PROXY_PIPE**: The `win-proxy` service must be operational on the host to bridge CDP traffic to the WSL artery.

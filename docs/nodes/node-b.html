Backend Notes

  • Vulkan only. ROCm support on consumer AMD cards (RDNA 3.5) is unreliable. Vulkan via ik_llama.cpp is the stable path.
  • TurboQuant q4_0 KV hurts on Vulkan. f16 KV cache only for Node B. This is a known ik_llama.cpp Vulkan limitation.
  • Flash Attention is functional and enabled.
  • llama.cpp build located at D:\\llama.cpp\\ on the Windows host (llama-server.exe ports 8081, 8082).
  • VRAM: ~10.4GB used of 16GB (Hermes 8.4GB + Qwen3-VL 1.9GB shared GPU).

Services

  • LiteLLM mesh router (Docker Desktop, port 4000) — deployed, 4 routes: mesh-fast, mesh-vision, mesh-heavy, mesh-function-calling
  • hermes-relay v0.6.1 (Docker Desktop, port 8767) — WSS bridge for mobile/desktop clients
  • Hermes TUI / Dashboard — active

Model Staging

All mesh models download to D:\llama.cpp\models\ (Windows) first, then SCP to target nodes. Node B can load directly from the Windows path via WSL mount at /mnt/d/llama.cpp/models/.

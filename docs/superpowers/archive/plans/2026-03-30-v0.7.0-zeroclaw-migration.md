# Project Black-Ice: v0.8.3 "Edge-Compute" Migration Plan
**Date:** March 30, 2026
**Target:** Node A (Rules Authority) & Node B (Orchestrator)
**Status:** SLATED (Post-v0.6.1 completion)

## 1. Objective
Decommission the high-overhead Docker/PostgreSQL stack on **Node A (192.168.0.50)** and replace it with a high-performance, Rust-native **ZeroClaw** implementation. Establish a sub-10ms **ClawLink** bridge to **Node B** and implement a **Unified Oracle** state manager on the main rig to eliminate narrative drift and hardware latency.

## 3. Implementation Phases

### **Phase 1: ZeroClaw Core (Rust)**
- **Implementation:** Scaffold `zeroclaw` Rust project using `rusqlite` and `sqlite-vec`.
- **Hybrid Search:** Register `sqlite3_auto_extension` for `sqlite-vec` and create the `vec0` Lore table.
- **Interlock Math:** Implement the deterministic `resolve_attack` and `calculate_dv` logic in Rust traits.

### **Phase 2: ClawLink Transport (SSH2)**
- **Client/Server:** Implement the socket listener on Node A and the client on Node B using the `ssh2` crate.
- **Persistence:** Establish a long-lived SSH session with `channel_direct_tcpip` for sub-10ms query times.
- **Corridor Security:** Implement Ed25519 token validation for all JSON-RPC frames.

### **Phase 3: Data Migration (Postgres → SQLite-Vec)**
- **Extraction:** Script the `COPY TO STDOUT` export of the 1,437 existing vector chunks from Node A Postgres.
- **Insertion:** Use `zerocopy` in Rust to perform high-speed binary insertion of `float32` vectors into `rules.db`.
- **Validation:** Verify 100% vector parity between old and new stores.

### **Phase 4: Unified Oracle (RKG on Node B)**
- **RKG Schema:** Create `world.db` with a Triplet schema (Subject, Predicate, Object).
- **Crush Sync:** Wire the `ATTACH DATABASE` logic to allow narrative grounding to join World State with Crush Session History.
- **HRC Update:** Update the `HybridRoutingController` to utilize the new `ClawLinkClient` instead of the Stdio bridge.

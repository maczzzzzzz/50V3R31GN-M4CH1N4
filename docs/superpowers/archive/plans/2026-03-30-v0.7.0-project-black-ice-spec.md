# Design Specification: Project Black-Ice (0.8.1)
**Date:** March 30, 2026
**Subject:** Rust-Native Edge-Compute Migration (ZeroClaw & ClawLink)
**Status:** DRAFT

## 1. Executive Summary
Project Black-Ice transitions the 50V3R31GN-M4CH1N4 from a "heavy" containerized architecture to a "lean" Rust-native edge-compute model. By replacing Docker/PostgreSQL with **ZeroClaw** (Rust/SQLite-Vec) and implementing the **ClawLink** persistent bridge, we reclaim Node A resources for deeper simulation and eliminate Node B narrative drift via a **Unified Strategic Strategic Strategic Strategic Strategic Strategic Oracle (RKG)**.

## 2. Technical Architecture

### 2.1 Node A: ZeroClaw (The Rules Authority)
- **Runtime:** Rust-native binary (~3.4MB).
- **Engine:** Built on `rusqlite` + `sqlite-vec`.
- **Artery of Truth:** `rules.db` (SQLite).
- **Capabilities:**
    - **Hybrid Search:** FTS5 (Keyword) + `vec0` (Semantic) search for lore and rules.
    - **Deterministic Math:** Rust-implemented Interlock mechanics (d10 + Stat + Skill).
    - **Zero-Tax:** <5MB RAM footprint (excluding model context).

### 2.2 The Mesh: ClawLink
- **Protocol:** Persistent Socket-over-SSH using the `ssh2` Rust crate.
- **Latency:** Target <10ms round-trip (currently ~100ms due to SSH-stdio startup).
- **Security:** Ed25519 public-key authentication for all Node B -> Node A requests.

### 2.3 Node B: Unified Strategic Strategic Strategic Strategic Strategic Strategic Oracle (Narrative Synthesizer)
- **Artery of Truth:** `world.db` (SQLite).
- **RKG (Relational Knowledge Graph):** Triplet-based storage (Subject-Predicate-Object).
- **Consolidation:** Uses SQLite `ATTACH DATABASE` to join `world.db` with `crush.db` (session history).
- **Grounding:** The Narrative Engine queries the RKG before every prose generation to ensure 100% factual consistency.

## 3. Data Migration Schema
- **Source:** PostgreSQL `pdf_chunks` table.
- **Target:** ZeroClaw `vec0` virtual table.
- **Vector Format:** `float32` blobs (binary) for maximum search speed.

## 4. Verification Plan
- **Parity Check:** 100% success rate on existing 237 tests using the new bridge.
- **Latency Stress:** 50 sequential queries must average <10ms response time.
- **VRAM Recovery:** Measure Node A RAM/VRAM delta after Docker decommissioning.

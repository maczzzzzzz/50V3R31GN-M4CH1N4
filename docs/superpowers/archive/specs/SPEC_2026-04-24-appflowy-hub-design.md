## 1. OBJECTIVE
To provide a structured, database-driven documentation and roadmap system that integrates with the Sovereign Identity and Obsidian RKG.

## 2. ARCHITECTURE (LOCAL-FIRST)
- **Backend:** AppFlowy Cloud (Rust-native) deployed via Nix on Node C.
- **Frontend:** Desktop client on Windows host + Web frontend via reverse proxy.
- **Aesthetic:** **Gruvbox Dark (Medium)**.
- **Data Artery:** Bi-directional Markdown sync for `IMPLEMENTATION_PLAN.md`.

## 3. INTEGRATION VECTORS
- **Profile Awareness:** Workspaces are tagged with profile metadata (`researcher`, `daily-use`).
- **GEPA Ingestion:** Trajectory documentation from GEPA loops are shored as AppFlowy database entries.

## 4. SUCCESS CRITERIA
- **Latency:** < 100ms for block-based editing.
- **Sovereignty:** 100% local storage. Zero external cloud dependency.

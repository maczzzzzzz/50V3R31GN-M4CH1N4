## Architecture Decisions

### Model Swap Protocol

The `PerceptionController` enforces sequential VRAM via a `tokio::sync::Mutex<VramGuard>`:

```
ocr_analyze called
  â†’ acquire vram_lock (blocks concurrent requests)
  â†’ POST /api/generate { keep_alive: 0 }  // unload Llama-3
  â†’ preprocess_image (base64 â†’ NCHW float32 tensor 1Ă—3Ă—224Ă—224)
  â†’ SessionBuilder::new(ort_env) + CUDA EP â†’ run inference â†’ drop Session
  â†’ ollama_warmup(".")                     // preemptive Llama reload
  â†’ release vram_lock â†’ return entities
```

### ONNX Runtime (ort 1.16)

- CUDA Execution Provider selected first, CPU fallback if unavailable
- `GraphOptimizationLevel::Level3` for Pascal sm_60 throughput
- Session is scoped and dropped within `run_falcon_inference` â€” VRAM freed immediately
- ORT `Environment` is constructed once at startup and shared via `Arc`

### Output Contract (Stub)

`decode_outputs` returns a `PERCEPTION_STUB` sentinel entity until Gemini deploys the Falcon ONNX model file and wires the real token decoder. This allows the full RPC pipeline to be validated end-to-end before the model file exists.

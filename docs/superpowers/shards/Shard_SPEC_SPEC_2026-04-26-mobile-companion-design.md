## 1. 🎯 THE HYBRID ARTERY
Our Flutter HUD does **NOT** eliminate the need for native apps on the phone. Instead, it acts as the **Tactical Overlay** for them.

## 2. 📱 THE TRIAD RELATIONSHIP
| App | Role | Interaction Pattern |
| :--- | :--- | :--- |
| **Obsidian (Android)** | Long-form reading / Canvas UI. | Reads the shared `data/vault/` directory synced via Tailscale/Syncthing. |
| **Logseq (Android)** | Block-level logging / Datalog queries. | Shares the same physical directory. Native mobile UI for high-speed input. |
| **Flutter HUD (Sovereign)**| Agentic Intent / Status / 3D Visualization. | **The Orchestrator.** Uses the Plugin Registry to send intents to the standalone apps or query their state via local sidecars. |

## 3. 🏗️ SHARED SHARD SYNC
The Flutter HUD manages the **Tailscale Artery**. It ensures that the physical files needed by Obsidian and Logseq are present and bit-identical to Node B.

## 4. 🔗 CONCLUSION
The native apps provide the **Human Rendering Engine**, while the Flutter HUD provides the **Agentic Cortex**. They lean on each other for a complete Sovereign Mobile experience.

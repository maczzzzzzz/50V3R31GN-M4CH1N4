## 1. Vision & Purpose
The **Nucleus Command Deck (CL4W)** is the singular unified interface for the 50V3R31GN-M4CH1N4 ecosystem. It decommissions all fragmented Rust EGUIs and TUIs in favor of a monolithic, high-fidelity WebGL command center. It serves as the **System Bootloader**, the **Orchestration Monitor**, and the **Final Authority** for the Virtual System Bus (VSB).

## 2. Visual & Auditory Sovereignty
- **UI Architecture:** Full-screen WebGL surface powered by **PIXI.js v8** and the **Pretext Engine**.
- **Aesthetic:** Total Red Shift (#ff003c), high-contrast black backgrounds, VT323 typography.
- **Atmosphere:** Master Sovereign Shroud shader (scanlines, aberration, purpose-driven tearing).
- **Governance Audio:** High-fidelity **Dial-Up Handshake Tone** loops during VSB Approval (Flush Gate) triggers.

## 3. Component Architecture: Unified Panels
The dashboard is organized into four holographic quadrants using 0-reflow Pretext rendering:
1. **`:/COMMAND //` (The REPL):** Direct 12B Brain interaction and narrative stream.
2. **`:/SENSORY //` (The Radar):** Real-time isometric grid for VSB blip tracking.
3. **`:/INTRUSION //` (The Deck):** Netrun depth, intrusion levels, and ST3GG status.
4. **`:/LOGISTICS //` (The Monitor):** VSB waveforms, Gauntlet results, and lifecycle bars.

## 4. Operational Control: The Nucleus Dropdown
A persistent top-center menu providing:
- **Ignition:** `[GHOST_BOOT]`, `[FULL_ENGAGE]`, `[LITE_MODE]`.
- **Hardware:** Force Reboot Node A, Toggle CUDA Shells, Unseal 7H3-V4UL7.
- **Governance:** Flush Gate modal with `[ACKNOWLEDGE] / [VETO]` and dial-up audio.

## 5. Self-Healing Artery & Autonomous Monitoring
The Nucleus Deck integrates high-fidelity hardware observability based on the **Hermes Monitoring Toolkit**:
- **Sovereign Pulse:** A cron-based background service that samples GPU/CPU telemetry from Node A and Node B every 60 seconds.
- **Risk Projection:** Real-time analysis of VSB integrity (Binary UDP packet loss) and Mmap state drift.
- **Vital Dashboard:** An ASCII-rendered `VITAL_SIGNS.md` manifest automatically updated in the Obsidian vault for zero-latency monitoring.

## 6. Shard Sovereignty & The Skill Factory
Every capability is a modular **Ability Shard** evolved through the **Hermes Skill Factory** patterns:
- **Shard Distillation:** Node B (Director) watches session logs to detect successful pattern repetitions, automatically proposing new `SKILL.md` bundles for Node A (Kernel).
- **Proposal Loop:** All new Shards must be human-approved via the Nucleus Deck before the Go Artery physicalizes them.
- **Identity Shroud (Icarus):** A persistent "Soul" layer that logs high-quality decision tagging into `SOUL.jsonl`. This ensures the project's "Identity" remains constant even during LLM model swaps.

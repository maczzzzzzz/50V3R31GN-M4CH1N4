Hermes Relay

Current Deployment: Native systemd user service on Node A

Purpose

Hermes Relay provides the messaging gateway layer, connecting external platforms (Telegram, Discord, Slack, etc.) to the Sovereign Mesh Hermes instances.

Current Topology

  • Node A (Synapse): Native hermes-relay (port 8767)
  • Node B (Director): Previously ran in Docker — now deprecated

Why Native on Node A?

  • Lower memory footprint than Docker
  • Better Tailscale integration
  • Consistent with native-first strategy (see Mesh Router)

Windows Laptop Support

Full installation instructions for new Windows devices are maintained in sidecars/hermes-relay/README.md.

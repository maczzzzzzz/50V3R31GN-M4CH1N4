Sovereign Mesh Router

Status: Active (May 2026)  |  Location: sidecars/mesh-router/

Overview

The Sovereign Mesh Router is a minimal, native, OpenAI-compatible proxy that routes requests to the correct inference node based on model name. It replaces the previous LiteLLM Docker deployment.

Design goals: Zero database, zero Prisma, no Docker networking complexity, fully native NixOS + systemd integration.

Architecture

  • Implementation: Single-file FastAPI application (~120 LOC)
  • Runtime: Python 3.12+ with FastAPI + Uvicorn + HTTPX
  • Deployment: User systemd service on Node B
  • Port: 4000 (localhost only)

Endpoints

  • GET /v1/models — Returns all available mesh models
  • POST /v1/chat/completions — Main inference endpoint (supports streaming)
  • GET /health — Simple health check

Service Management

# Check status
systemctl --user status mesh-router

# Restart
systemctl --user restart mesh-router

# View logs
journalctl --user -u mesh-router -f

Why Not LiteLLM?

The previous LiteLLM deployment (Docker + ghcr.io/berriai/litellm:main-latest) suffered from repeated Prisma migration failures and fragile Docker Desktop networking on WSL2. The native router eliminates both classes of problems while providing only the features the mesh actually needs.

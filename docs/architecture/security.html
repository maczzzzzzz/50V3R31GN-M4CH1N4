Zero-Trust Model

The mesh operates on zero-trust principles. No node implicitly trusts another. All authentication is handled by Tailscale identities and SSH key pairs. There are no shared passwords or service tokens.

NixOS Safety Assertions

The shared NixOS config (nix/hosts/shared/common.nix) includes a hard assertion that prevents accidental lockout:

assertions = [{
  assertion = config.wsl.enable -> config.wsl.defaultUser == "nixos";
  message = "BLOCKED: wsl.defaultUser must be 'nixos'.
    Changing it breaks WSL auto-login and locks out the primary user.";
}];
This assertion was added after an autonomous agent changed wsl.defaultUser to a wrong value during an overnight session, locking out the primary user on Node B. The assertion ensures no future rebuild can succeed with a wrong default user.

Tailscale ACL

Access control is managed at the Tailscale admin console. The current policy:

  • SSH: autogroup:members may SSH to autogroup:self as nonroot or root
  • Default allow-all for internal traffic (single-user tailnet)
  • No exit nodes configured

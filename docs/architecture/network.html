Zero-Trust Artery

All inter-node communication flows through Tailscale (WireGuard). Each node authenticates via its Tailscale identity. There is no trusted LAN — every connection is encrypted and authenticated regardless of physical network topology.

SSH Configuration

SSH is configured for headless key-based access across the mesh. The Tailscale ACL includes an explicit SSH policy:

{
  "ssh": [{
    "action": "accept",
    "src": ["autogroup:members"],
    "dst": ["autogroup:self"],
    "users": ["autogroup:nonroot", "root"]
  }]
}

All nodes run sshd with PasswordAuthentication = false (except Node C during initial setup). Users connect via ssh nixos@<tailscale-ip> without browser-based auth challenges.

Known Issues

  • Node C loses its local IP on reboot (DHCP, no static reservation). MAC address needs DHCP reservation on the router or static NixOS config.
  • Tailscale version mismatch: WSL client at 1.96.5, nodes at 1.80.3. Not currently causing issues but should be aligned.
  • Node C Tailscale service not in NixOS config — installed manually. Needs to be added to the flake config.

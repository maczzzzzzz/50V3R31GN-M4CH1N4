◈ PLUGIN ARCHITECTURE & MANIFESTO

Category: ENGINE | Status: MANDATED

The No-Shadow Manifesto

Nodestadt plugins must adhere to the No-Shadow Logic mandate. We utilize native Hermes v0.13.0 hooks to inject Sovereign capabilities rather than building custom shim layers.

  • Modularity: Each appendage must be a self-contained Python or Rust module.
  • Auditability: Every tool call must be observable by the Psy-core hook.
  • Parity: Standardized deployment via Nix flakes across all nodes.

2. Structural Standards

Manifest: plugin.yaml

Every plugin must include a manifest defining its name, type, and hardware requirements.

name: resilience-forge
type: general
version: 1.0.0
tags: [security, vaccination]

Logic Core: __init__.py

Initialization must be non-blocking. Heavy computation should be deferred to background threads or specialized nodes (Node D).

3. MCP (Model Context Protocol)

High-level appendages are materialized as MCP servers. This allows for language-agnostic tool expansion (Rust, Node.js, Go) while maintaining bit-identical compatibility with the Hermes core.

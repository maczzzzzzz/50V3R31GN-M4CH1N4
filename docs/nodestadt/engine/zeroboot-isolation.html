◈ ZEROBOOT ISOLATION

Category: ENGINE | Status: MATERIALIZED

Hardware Sandbox Layer

Zeroboot provides Firecracker/KVM based microVM isolation for non-deterministic agent code execution. It ensures that any "execute_code" or untrusted shell tasks are physically contained on Node D.

1. Technical Architecture

  • Host: Node D (Intel Core Ultra).
  • Logic: crates/modules/zeroboot-isolation/ (Rust Manager).
  • Nix Module: nix/modules/zeroboot.nix.

2. VM Lifecycle

The Zeroboot manager handles sub-100ms cold starts for fresh execution environments.

  • Provision: Fetches a read-only rootfs image from Node A.
  • Execute: Mounts task data and triggers the Firecracker process.
  • Reclaim: Destroys the VM state immediately upon task completion.

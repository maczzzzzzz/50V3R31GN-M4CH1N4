◈ VIRTUAL SOVEREIGN BUS (VSB)

Category: ENGINE | Status: OPERATIONAL

The Cognitive Heartbeat

The VSB Protocol is a native ModelProvider synchronization that synchronizes model availability and load status across the mesh. It enables the agent to treat four physical nodes as a single, unified reasoning surface.

1. Native ModelProvider Integration

Hermes v0.13.0 API Compliance

The VSB is now implemented as a native ModelProvider plugin. It utilizes persistent TCP streams over the Tailscale Artery, eliminating the need for legacy UDP pulses.

  • Transport: HTTP/2 (Persistent gRPC/REST).
  • Auth: Zero-Trust MTLS (Tailscale).
  • Health: Real-time KV-cache availability via Tenacity heartbeat.

2. Dynamic Model Routing

The sovereign_vsb provider performs real-time dispatch based on node health and model specialization.

Specialized Tiers

  • HEAVY: Routed to Node D (Qwen 3.6 35B MoE (Strategic Core)) for deep architectural reasoning.
  • STRATEGY: Routed to Node B (Qwen 3 14B (Workspace Authority)) for planning and vision.
  • TRIAGE: Routed to Node C (Qwen 3.5 0.8B (Perception)) for high-speed perception.

3. Backend Integration

All nodes must run an OpenAI-compatible llama-server or vLLM instance on port 8080 or 9119. The VSB router maps these local endpoints to the Tailscale Artery.

# config.yaml Snippet
model_providers:

    type: model
    config:
      mesh_nodes:

          ip: 100.120.225.12
          port: 8080

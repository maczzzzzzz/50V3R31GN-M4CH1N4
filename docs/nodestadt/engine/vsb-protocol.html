◈ VIRTUAL SOVEREIGN BUS (VSB)

Category: ENGINE | Status: OPERATIONAL

The Cognitive Heartbeat

The VSB Protocol is a high-frequency binary UDP pulse that synchronizes model availability and load status across the mesh. It enables the agent to treat four physical nodes as a single, unified reasoning surface.

1. UDP Pulse Specification

Packet Structure (302-byte Deterministic)

  • Bytes 0-2: Header (ASCII 'VSB').
  • Byte 3: Protocol Version (Current: 3).
  • Byte 4: Node Identifier (Int).
  • Bytes 5-8: Load Average (Float32).
  • Bytes 9-12: RAM Usage (Float32).
  • Bytes 13-16: VRAM Usage (Float32).
  • Bytes 17-269: Padding (Zero-filled).
  • Bytes 270-301: Cryptographic HMAC-SHA256 (Secret-keyed).

Port: 7878 | Frequency: 2.0s intervals.

2. Dynamic Model Routing

The sovereign_vsb provider performs real-time dispatch based on node health and model specialization.

Specialized Tiers

  • HEAVY: Routed to Node D (Qwen2.5-Coder-14B) for deep architectural reasoning.
  • STRATEGY: Routed to Node B (Carnice-9B) for planning and vision.
  • TRIAGE: Routed to Node C (Qwen3.5-0.8B) for high-speed perception.

3. Backend Integration

All nodes must run an OpenAI-compatible llama-server or vLLM instance on port 8080 or 9119. The VSB router maps these local endpoints to the Tailscale Artery.

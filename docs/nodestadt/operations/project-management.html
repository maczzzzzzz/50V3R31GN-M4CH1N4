◈ MESH PROJECT MANAGEMENT

Category: OPERATIONS | Status: OPERATIONAL

The Distributed Ledger

Nodestadt utilizes an embedded Kanban Dispatcher within the Hermes Gateway to manage multi-node missions. Task state is synchronized across the mesh via the kanban.db ledger on Node B.

1. Mission Workflow

State Definition Transition
Triage Initial assessment of intent. → Todo
Ready Task queued for worker claim. → Running
Running Actively processed by an agent. → Done / Blocked
Blocked Halted due to error or dependency. → Ready

2. Agent Worker Lifecycle

  1. Poll: The dispatcher (Node B) identifies ready tasks.
  2. Claim: A worker process is spawned with HERMES_KANBAN_TASK context.
  3. Heartbeat: The agent issues heartbeats every 5 minutes to maintain claim lock.
  4. Finalize: Task is marked done and result is archived in the session ledger.

3. Command Reference

# View mesh-wide tasks
hermes kanban list

# Inspect specific mission
hermes kanban show [task_id]

# Create new coordination task
hermes kanban create --title "Bridge MATLAB" --status ready

4. Artery Notifications

Mission events (Completion, Failure, Blockers) are automatically pushed to the Messaging Artery (Telegram/Discord) for real-time human awareness.

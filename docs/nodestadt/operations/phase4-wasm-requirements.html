Phase 4 WASM Build Requirements

WASM Build Dependencies

The Phase 4 Pretext HUD implementation includes WASM bindings for high-performance text layout in the browser. The following dependencies are required for building the WASM module:

Required Tools

  • wasm-pack: Rust-to-WASM build tool
    cargo install wasm-pack
  • lld linker: LLVM linker for WASM
    sudo apt install lld (Debian/Ubuntu)
    or install via Nix: nix-env -iA nixpkgs.lld

Build Instructions

# From project root
cd crates/modules/pretext-core

# Build WASM package
wasm-pack build --target web --out-dir ../../dashboard/hermes-workspace/wasm/pkg

# This generates:
# - pkg/pretext_core.js (JavaScript bindings)
# - pkg/pretext_core_bg.wasm (WASM binary)
# - pkg/pretext_core.d.ts (TypeScript definitions)

Current Implementation Status

✅ Complete:

  • Pretext Core Rust engine with all layout arithmetic
  • ASCII mapper with gradient, wave, and pulse effects
  • Thought-stream virtualization for predictive height measurement
  • React usePretext hook with fallback Dart implementation
  • Flutter/FFI bindings for mobile Machina Terminal
  • Fluid smoke shader for Ambient Artery background

⚠️ Known Limitation:

The React components (usePretext.ts, KineticThoughtStream.tsx) include fallback implementations that work without WASM. The WASM bindings provide performance benefits but are not strictly required for functionality.

Testing Without WASM

The dashboard will run in "Dart fallback mode" when the WASM module is not available. This uses a pure JavaScript implementation of the layout arithmetic that is functionally equivalent but slightly slower than the WASM version.

The Two-Repo Model

Sovereign Machina utilizes a nested repository architecture to maintain absolute control over the logic stream while remaining compatible with rapid upstream releases.

1. Daily Development Workflow

Work always occurs in the monorepo root. Changes to sidecars are committed to the fork's main branch and then "pinned" in the monorepo.

Modifying a Sidecar

  1. Enter Submodule: cd sidecars/hermes-agent-nous
  2. Commit Logic: git add . && git commit -m "feat: ..." && git push origin main
  3. Bump Pin: Return to monorepo root and commit the submodule pointer change.

2. Upstream Synchronization

We maintain parity with upstream sources (NousResearch, BasedHardware) by merging their main into our Sovereign forks.

# Standard Sync Protocol
cd sidecars/hermes-agent-nous
git fetch upstream
git merge upstream/main -m "merge: sync with upstream"
git push origin main

3. Engineering Discipline

  • No Shadow Logic: Custom patches must be modular and documented.
  • Nix Reproducibility: All environment changes must be reflected in flake.nix.
  • Semantic Versioning: Automated via semantic-release across the mesh.

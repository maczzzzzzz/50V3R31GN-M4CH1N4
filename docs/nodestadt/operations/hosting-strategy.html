◈ DOCUMENTATION HOSTING STRATEGY

Category: OPERATIONS | Status: PROPOSED

The Distributed Library

To ensure our Semantic HTML documentation remains accessible and high-fidelity, we will deploy a multi-layered hosting strategy. This bypasses the limitations of Markdown-native platforms like GitBook.

1. Primary Options

Platform Role Sync Method
GitHub Pages Public Manual Native /docs folder sync.
Netlify / Vercel Staging/Previews GitHub Action on PR.
Sovereign Caddy Internal Artery Rsync to Node A (Synapse).

2. The Public Artery: GitHub Pages

The most direct replacement for GitBook. It serves our raw .html and .css files directly from the monorepo.

  • Endpoint: nodestadt.github.io/50V3R31GN-M4CH1N4/
  • Setup: Settings > Pages > Build from /docs folder on beta/v3.
  • Benefit: Zero-cost, native versioning, supports custom CSS.

3. The Physical Artery: Node A (Synapse)

For absolute sovereignty, the documentation hub is hosted on Node A (100.90.196.70) and accessed via the Tailscale Artery.

Internal Deployment

  1. Service: Caddy server running on Node A.
  2. Path: /var/www/nodestadt-docs
  3. Update: Automated rsync from Node B (Director) during the stabilization cycle.

4. Recommendation

Deploy to GitHub Pages for public developer onboarding and Node A for internal mission coordination. This ensures that even if GitHub is unreachable, the mesh remains self-documented.

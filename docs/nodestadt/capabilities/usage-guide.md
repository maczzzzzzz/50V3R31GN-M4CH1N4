## ◈ 1. THE INTERFACE (PRETEXT HUD)

The HUD is divided into tactical modules.
- **Left Rail (Vitals):** Real-time monitoring of VRAM saturation across the mesh. If any node turns RED, research swarms will automatically downscale.
- **Center Canvas (Hermes):** Your primary reasoning interaction point.
- **Right Rail (Shards):** Active implementation tasks and memory triplets.

### ◈ Command Hotkeys
- `ctrl+i`: Ignite/Restart full mesh.
- `ctrl+p`: Force-purge zombie processes.
- `shift+q`: Graceful quaternary shutdown.

## ◈ 2. AGENTIC REASONING (HERMES INK)

Use the `/` prefix for direct system commands inside the terminal.

### ◈ Research & Materialization
- `/research [topic]`: Dispatches Node D for exhaustive document scans. Use this for architectural audits.
- `/forge [asset]`: Converts reasoning trajectories into permanent Skills/MCP tools.
- `/datalog [query]`: Directly query the Symbolic Core (RKG) using Datalog syntax.

### ◈ Governance Loop
When an agent proposes a change, you will see a **Proposal Shard** in the HUD.
- Use `/pass [id]` to approve.
- Use `/veto [id]` to reject.
- The **Rules Strategic Oracle** on Node A will automatically block trajectories that violate physical invariants.

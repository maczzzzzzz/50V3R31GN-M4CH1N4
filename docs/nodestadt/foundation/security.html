◈ SECURITY POLICY & TRUST MODEL

Category: FOUNDATION | Status: ENFORCED

1. Sovereign Trust Model

Nodestadt operates as a personal nervous system with a single trusted operator. The architecture prioritizes protecting the operator from rogue LLM actions while ensuring that mesh-wide communication is cryptographically secure.

  • Single Operator: Multi-user isolation is handled at the OS/Kernel level via Zeroboot.
  • Artery Encryption: All node-to-node traffic is WireGuard-encrypted via Tailscale.
  • Zero-Trust: Local network proximity does not imply authorization.

2. Structural Boundaries

Command Gating

All destructive terminal and file operations are gated behind explicit user confirmation. Autonomy levels are configurable per-session.

Secret Redaction

Sensitive patterns (API keys, tokens, session IDs) are surgically stripped from display output and logs before reaching the TUI or HUD interfaces.

Zeroboot Isolation

Untrusted workloads (e.g., executing scraped code) are routed to Node D for execution within sterile KVM/Firecracker microVMs.

3. Physical Hardening Mandates

  1. Credential Storage: All mesh secrets must reside in ~/.hermes/.env with chmod 600 permissions.
  2. Network Silence: No public ingress permitted. All interaction must occur via the Tailscale Artery.
  3. Audit Logging: All tool calls are intercepted by Psy-core and logged to a write-only append ledger.

Transport Layer Overview

The Artery defines the encrypted pathways through which the Nodestadt nodes maintain state synchronization and command authority. We utilize Tailscale to provide a secure, peer-to-peer overlay network across Nodes A, B, C, and D.

  • Encryption: WireGuard-based Zero-Trust transport.
  • Addressing: Statically mapped 100.x.y.z IPs.
  • Persistence: Immune to local LAN fluctuations.
  • Mobile Bridge: Enables Clawlink Android-to-Mesh connectivity.

1. VSB UDP Protocol (The Pulse)

Real-time logic synchronization and latency-critical telemetry are handled via the Virtual Sovereign Bus (VSB).

  • Protocol: Pure binary UDP on port 7878.
  • Packet Size: Fixed 302-byte packets for deterministic mesh processing.
  • Function: High-frequency state updates and model routing heartbeat.

2. mTLS Control Plane

All high-level tool calls and node-to-node RPCs are routed through an encrypted mTLS layer, ensuring that even if a local node is compromised, the mesh remains resilient.

  • Authority: hermes-router on port 7341.
  • Gating: Access is strictly governed by Psy-core security audits.
  • Identity: SPIFFE/SPIRE enforced per-session.

3. Model Storage & Sync

To maintain bit-identical reasoning, LLM weights are managed via a centralized distribution system.

  • Source: Physical Windows D: drive (Node B).
  • Distribution: rsync via Tailscale Artery to worker nodes.
  • Integrity: Mandatory SHA-256 verification after every transfer.

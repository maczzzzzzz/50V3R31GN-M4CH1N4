✅ FULL ANALYSIS – LiteSeek TokenSpeed (https://github.com/lightseekorg/tokenspeed + Blog Post)

Project Overview
TokenSpeed (marketed as LiteSeek TokenSpeed) is a new, high-performance LLM inference engine specifically optimized for agentic workloads (long-context, multi-turn, tool-calling, coding agents).
It aims to deliver “speed-of-light” inference with TensorRT-LLM-level performance and vLLM-level usability.

Core Technical Highlights (from repo + blog)
- Modeling Layer: Local-SPMD design with static compiler that auto-generates collective communication from module-boundary placement annotations.
- Scheduler: C++ control plane + Python execution plane. Request lifecycle, KV cache ownership, and overlap timing are encoded as a finite-state machine with type-system-enforced safety.
- Kernels: Pluggable layered kernel system with a centralized registry. Strong focus on MLA (Multi-head Latent Attention) kernels for agentic models (especially on Blackwell).
- Entrypoint: SMG-integrated AsyncLLM for low-overhead CPU-side request handling.
- Performance Claims: On Blackwell GPUs, it shows strong gains over TensorRT-LLM in agentic coding workloads (higher TPM/GPU at 70+ TPS/User targets).
- Status: Preview release (May 6, 2026). Heavy development ongoing. Not yet production-hardened.

Usefulness for Sovereign Machina (Beta v3 Context)

High Potential — This is a very relevant inference engine for our use case.

Strengths
- Agentic Optimization: Explicitly designed for the exact workload we care about (long-context coding agents, tool calling, multi-turn reasoning). This aligns perfectly with our live coding partner loop in Warp and heavy tasks on Node D.
- Performance on Consumer Hardware: Strong focus on Blackwell and Hopper, but the design (pluggable kernels, scheduler FSM, local-SPMD) should translate well to our AMD + NVIDIA mix (especially Node B 9060 XT and Node D iGPU).
- Modularity: Pluggable kernels and scheduler make it easy to integrate as a pluggable backend in our model router.
- Sovereign Fit: Fully local, open-source, no cloud. We can build it as a Nix package and run it inside Zeroboot sandboxes for isolation.

Risks / Limitations
- Very New: Preview release only. Production hardening is still in progress.
- Blackwell Focus: Best gains shown on Blackwell GPUs. We will need to test heavily on our 9060 XT and RTX 2060.
- Python/C++ Mix: Core scheduler in C++, execution in Python — we can wrap it but it adds another runtime layer.
- Maintenance: Active development means frequent changes. We would pin a specific commit in our monorepo (standard submodule practice).

Recommended Integration for Beta v3
- Add TokenSpeed as an optional pluggable inference backend in the monorepo (crates/modules/tokenspeed-backend/).
- Update the model router to choose TokenSpeed for agentic/coding-heavy tasks on supported nodes.
- Run it inside Zeroboot for isolation and Psy-core audit.
- Keep mainline llama.cpp / vLLM as fallback for broader model support.

This gives us a high-performance agentic inference option without making it the default for everything.

Verdict
Strong usefulness as an optional high-performance backend for agentic workloads.
It fits our “lean into strong external tools and layer our sovereign optimizations” direction for beta v3.
We should evaluate it during the Phase 122 repo sweep and add it as a pluggable module if benchmarks on our hardware confirm the gains.

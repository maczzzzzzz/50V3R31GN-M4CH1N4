Droid CLI operates on a strict JSON hierarchy. All configuration for the Quaternary Mesh must be explicitly defined.

  • Global Config: ~/.factory/settings.json (User-level overrides, Autonomy level)
  • Project Config: 50V3R31GN-M4CH1N4/.factory/settings.json (Repo-level rules)
  • Local Overrides: 50V3R31GN-M4CH1N4/.factory/settings.local.json (Ignored by git, useful for Node B specific paths)

2. Tribal Knowledge (AGENTS.md)

Droid inherently looks for AGENTS.md in the current directory and cascades upward. Our existing AGENTS.md perfectly satisfies this pattern, providing the strict branch and repository tracking mandates to GLM-5 before it touches code.

3. Custom Droids (Subagent Delegation)

Droid supports localized subagent delegation, mapping directly to our transient worker architecture.

  • Location: .factory/droids/<name>.md
  • Syntax: YAML Frontmatter + Markdown.
  • Integration: GLM-5 can spawn these specialized droids for isolated tasks.

Example: Rust Auditor Subagent (.factory/droids/rust-auditor.md)

name: rust-auditor
description: Audits Rust code against the Psy-core cryptographic standards.

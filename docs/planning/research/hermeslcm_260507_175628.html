✅ DEEP ANALYSIS & INTEGRATION SPEC – hermes-lcm (Lossless Context Management Plugin)

Repo: https://github.com/stephenschoettler/hermes-lcm
Status: Actively maintained (latest release v0.9.1 as of May 7, 2026). MIT license.

Core Purpose
hermes-lcm is a lossless context management plugin for Hermes Agent. It stores every raw message in an SQLite database (lcm.db) and builds a hierarchical summary DAG (Directed Acyclic Graph) to keep the active context bounded while allowing perfect recall of any prior message or summary. It never throws away data — it compresses intelligently and provides agent tools to retrieve or expand context on demand.

Key Features Relevant to Us - SQLite + FTS5 for raw message storage. - DAG-based summarization: Messages are grouped into depth-aware summary nodes (D0 leaves → higher-depth parents). Older context is compacted without loss. - Tools for the agent: lcm_grep, lcm_load_session, lcm_describe, lcm_expand, lcm_expand_query, lcm_status, lcm_doctor. - Session controls: Ignore noisy sessions or mark them as stateless. - Large-output handling: Externalization of oversized tool results. - Diagnostics: Slash commands and doctor repair tools.

Architecture
- Ingest → Store raw → Compact into DAG → Assemble bounded context for the prompt. - Tools allow the agent to query the DAG intelligently.

FIT WITH OUR BETA V3 REBUILD STRATEGY

This plugin is an excellent fit and aligns perfectly with our decision to lean heavily into Hermes-native capabilities while layering our sovereign optimizations as pluggable modules.

  • It directly addresses lossless context management — a core weakness in most agent systems.
  • It uses a DAG for hierarchical summarization, which complements our MemPalace graph structure.
  • It can serve as the base memory layer for Hermes, with our GEPA v2 + HALO + Ouroboros loop + weighted healing layered on top as extensions.
  • It is modular by design and can be enabled/disabled without breaking the core system.

How It Layers on Our Existing Components

  • Hermes Base Memory → LCM becomes the primary storage and summarization engine.
  • MemPalace Graph → We keep our graph retrieval and Mooncake KV spillover as a pluggable module on top of LCM (for 3D visualization and advanced queries).
  • GEPA v2 + HALO → GEPA can use LCM tools for context expansion and reflection. HALO can analyze LCM run history for trace optimization.
  • Ouroboros Loop / Weighted Learning → We can extend LCM with our weighted healing logic (e.g., score summaries and decide when to escalate depth or prune).
  • Psy-core → Add tamper-proof audit to the LCM SQLite database (hash chain on every message and summary node).

This gives us the best of both worlds: Hermes-native lossless management + our sovereign graph, healing, and audit optimizations.

CONCRETE INTEGRATION PLAN (Monorepo-First)

1. Treat hermes-lcm as a Pluggable Module
Create crates/modules/hermes-lcm-integration/ in the monorepo (tracked directory, not a submodule).

2. Nix Module for Easy Enable/Disable

services.hermes-lcm = {
  enable = true;           # default on for beta v3
  sqlite_path = "/var/lib/hermes/lcm.db";
  context_threshold = 0.75;
  fresh_tail_count = 50;
  psy_core_audit = true;   # our sovereign layer
};

3. GEPA Extension (Optional Module)

// crates/modules/gepa-core/src/extensions/lcm.rs
pub struct LCMExtension {
    enabled: bool,
}

impl GEPAExtension for LCMExtension {
    fn enhance_reflection(&self, reflection: &mut Reflection, context: &MemPalaceContext) {
        if !self.enabled { return; }
        let expanded = lcm::expand_context(&reflection.query, context).await?;
        reflection.apply_expanded_context(expanded);
    }
}

4. Pretext HUD Panel – “LCM Context” Add a panel showing DAG status, active summaries, and recall quality.

5. Warp Skill Support Extend the sovereign-hermes skill with /lcm commands that forward to Hermes-native LCM tools.

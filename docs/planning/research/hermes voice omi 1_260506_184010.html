✅ FULL INTEGRATION SPEC – Leveraging Hermes Stock Voice + VibeVoice + VoxCPM2 + Omi Hardware → Mobile App

Locked Hierarchy
- Pretext HUD = central home screen / OS hub
- Hermes (patched fork submodule) = authoritative runtime and orchestrator
- Warp = specialized dev appendage
- Mobile app (Flutter terminal-app) = voice + device trigger appendage

We will not replace Hermes voice mode. We will amplify it using VibeVoice (ASR) and VoxCPM2 (TTS), while keeping Omi as the hardware input layer.

1. HIGH-LEVEL ARCHITECTURE

Voice Flow
Omi hardware (wearable / Android app) → raw audio / partial transcript → Clawlink bridge (on Node D) → Hermes Voice Mode (Whisper STT) → VibeVoice ASR enhancement (structured output, diarization, long-form) → Hermes Agent processing → VoxCPM2 TTS (48kHz, expressive, cloning) → audio back to Omi app + logged in MemPalace.

Key Benefits
- Hermes provides the agentic core and native voice commands.
- VibeVoice adds high-quality, structured, multi-speaker ASR.
- VoxCPM2 adds studio-quality, controllable, multilingual TTS with voice design/cloning.
- Omi provides the physical hardware input.
- All tied into the Flutter mobile app via MCP/REST.

Integration Point
All new logic lives in the monorepo as pluggable modules under crates/modules/ or sidecars/. No changes to the Hermes submodule unless a sovereign patch is needed.

2. CONCRETE INTEGRATION PLAN

A. Hermes Stock Voice Mode (Base Layer)
Already enabled in the patched fork.
Use the native voice mode (Whisper STT + TTS) as the starting point.

Configuration (in the Hermes fork, cli-config.yaml or ~/.hermes/config.yaml):

voice:
  record_key: "ctrl+b"
  auto_tts: true
  stt:
    provider: "local"
    local:
      model: "large-v3"   # or base for speed
  tts:
    provider: "edge"      # temporary, we will override with VoxCPM2

B. Amplify STT with VibeVoice (Microsoft)
VibeVoice-ASR-7B gives structured output (speaker, timestamps, content), long-form support, and multilingual capability.

New Module: crates/modules/vibevoice-asr/

Integration (in monorepo): - Run VibeVoice locally via vLLM or Transformers. - Pipe Hermes Whisper output into VibeVoice for post-processing (diarization, structuring, noise robustness).

C. Amplify TTS with VoxCPM2 (OpenBMB)
VoxCPM2 gives 48kHz studio-quality, voice design, controllable cloning, and real-time streaming.

New Module: crates/modules/voxcpm-tts/ (already started in previous discussions)

Integration: - Replace Hermes default TTS with VoxCPM2 calls. - Support voice design prompts and cloning from Omi reference audio.

D. Omi Hardware Bridge
Use the existing Omi MCP server + Clawlink bridge (already in monorepo) to send raw audio or partial transcripts from the Omi app/wearable to the Node D voice pipeline.

E. Flutter Mobile App Tie-In (terminal-app/)
- Extend the Flutter app to send voice data via WebSocket/MCP to the Clawlink bridge on Node D.
- Receive TTS audio back from VoxCPM2 and play it locally in the app.

3. CONCRETE IMPLEMENTATION (Monorepo)

1. New Voice Pipeline Module (in monorepo)

Create crates/modules/voice-pipeline/

Cargo.toml

[dependencies]
sovereign-mcp = { path = "../../crates/mcp" }
sovereign-memory = { path = "../../crates/memory" }
vibevoice = { git = "https://github.com/microsoft/VibeVoice" }   # or local path
voxcpm = { git = "https://github.com/OpenBMB/VoxCPM" }

Main Entry

#[mcp_tool]
async fn process_voice(
    audio: Vec<u8>,
    context: MemPalaceContext,
) -> Result<VoiceResult> {
    // 1. Hermes Whisper STT (base)
    let transcript = hermes_voice::transcribe(&audio).await?;

    // 2. Amplify with VibeVoice ASR
    let structured = vibevoice::enhance_asr(&transcript, &audio).await?;

    // 3. Hermes Agent processing
    let response = hermes::process(&structured.text).await?;

    // 4. Amplify TTS with VoxCPM2
    let audio_out = voxcpm::generate_tts(&response, voice_prompt = structured.style).await?;

    // 5. Log to MemPalace + Psy-core audit
    let episode = DevEpisode::from_voice_interaction(&structured, &response);
    context.graph.store_episode(episode).await?;
    psy_core::audit_voice(&episode).await?;

    Ok(VoiceResult { transcript: structured.text, audio: audio_out })
}

2. Nix Module for Voice Pipeline

services.voice-pipeline = {

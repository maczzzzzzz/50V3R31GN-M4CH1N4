✅ FULL ANALYSIS – Graphify (https://github.com/safishamsi/graphify)

Repo Status (verified live as of 2026-05-07)
- License: MIT (fully open and compatible with our sovereign goals).
- Activity: Extremely high — 42.2k stars, frequent commits (last commit within the last few days), active community.
- Primary Language: Python (100%).
- Purpose: A graph-RAG / knowledge-graph plugin for AI coding assistants (Claude Code, Cursor, Gemini CLI, Hermes, OpenClaw, Factory Droid, etc.).
It turns any folder of code, docs, PDFs, images, videos, or URLs into a queryable, structured knowledge graph that the assistant can use instead of grepping files.

KEY FEATURES & HOW IT WORKS

Core Workflow
1. Run /graphify . (or graphify extract) in the assistant.
2. Graphify parses the folder:
- Code → Tree-sitter AST (local, no LLM for code).
- Non-code (docs, PDFs, images, video, URLs) → LLM semantic extraction.
3. Builds a knowledge graph (nodes = concepts, edges = relationships, confidence tags: EXTRACTED / INFERRED / AMBIGUOUS).
4. Outputs:
- graph.html (interactive visualization).
- GRAPH_REPORT.md (summary, god nodes, surprising connections, suggested questions).
- graph.json (full graph for programmatic use).
5. The assistant automatically uses the graph for future queries (via hooks or MCP server).

Advanced Capabilities
- Multi-modal ingestion — code, Markdown, Office docs, PDFs, images, video/audio (local transcription), YouTube URLs.
- MCP Server — Exposes the graph via standard MCP tools (query_graph, get_node, get_neighbors, shortest_path).
- Global Graph — Register multiple project graphs into ~/.graphify/global.json for cross-project queries.
- Hooks & Auto-rebuild — Git post-commit hooks rebuild the graph automatically (AST-only, low cost).
- Privacy-first — Code parsed locally; only non-code assets sent to the LLM backend (configurable: Gemini, Claude, Ollama, etc.). No telemetry.

USEFULNESS FOR SOVEREIGN MACHINA (BETA V3 CONTEXT)

High Usefulness — This is a strong, practical addition to our memory layer and agentic workflows.

Direct Benefits
- Memory Layer Enhancement: Graphify provides a structured, queryable knowledge graph of codebases, docs, and multi-modal assets. This complements our MemPalace graph perfectly — we can ingest project folders into Graphify and store the resulting graph.json in MemPalace for persistent recall.
- GEPA / Agentic Workflows: Agents can query the graph instead of raw files → fewer tokens, better structural reasoning, and more accurate context. This strengthens the live coding partner loop in Warp.
- Hermes Ecosystem Fit: Has native Hermes support (dedicated installer and MCP integration). It slots directly into our Hermes-first strategy.
- Pretext HUD / Visual Memory Palace: The interactive graph.html and graph.json can be surfaced as a new panel or rendered in 3D in Visual Memory Palace.
- Beta v3 Modularity: Perfect as a pluggable module in the monorepo (crates/modules/graphify-integration/). We can run it inside Zeroboot for isolation, audit it with Psy-core, and enable/disable it via Nix options.

Potential Use Cases in Our Stack
- Auto-build knowledge graphs for any project folder on git commit (via hook).
- Query the graph from Warp (/graphify query "what connects auth to database?").
- Use in GEPA reflections for structural understanding of codebases.
- Cross-project global graph for research or multi-repo work.
- Visual Memory Palace nodes that link to Graphify graphs for richer 3D memory.

Risks / Limitations
- Python-based — we will run it inside Zeroboot or as a thin Rust wrapper for isolation.
- LLM calls for non-code extraction — configurable to use our local models.
- Not a full replacement for our custom MemPalace graph — it is a complementary tool.

Verdict
High usefulness. Graphify is a mature, actively maintained graph-RAG plugin that directly strengthens our memory layer and agentic capabilities. It fits our “lean into Hermes ecosystem” direction for beta v3 while adding real value without duplicating effort.

RECOMMENDED INTEGRATION (Beta v3)

Module: crates/modules/graphify-integration/ (tracked directory in monorepo).

Nix Config

services.graphify = {
  enable = true;
  zeroboot_sandbox = true;
  auto_rebuild_on_git_commit = true;
  mcp_enabled = true;
};

Hermes MCP Tool Wrapper

#[mcp_tool]
async fn graphify_query(
    query: String,
    project_path: String,
    context: MemPalaceContext,
) -> Result<GraphQueryResult> {
    let result = graphify::query(project_path, query).await?;
    context.store_graph_query(&result).await?;
    Ok(result)
}

After-Action Report

Rescue Operation: Tailscale Artery & Quaternary Identity Alignment

1. Incident Summary

Between May 7th and May 9th, 2026, the NODESTADT mesh experienced a critical connectivity and identity failure. The Zero-Trust Artery (Tailscale) was blocked by physical network drops, NixOS configuration drift, and a "Ghost Identity" mismatch.

2. Identified Failure Modes

  • Identity Fracture: The logical configuration defined the nixos user, while the physical nodes were managed by maczz. Each nixos-rebuild attempted to "sterilize" the active user, leading to session kills and D-Bus failures.
  • Filesystem Instability: Missing Btrfs subvolume mappings for /home caused NixOS to attempt unmounting the active user's directory during system switches.
  • Trust Blockage: Closure signature mismatches prevented Node B from pushing updates to Node A and C, requiring a shift to remote build-host strategies.
  • Physical Link Drop: Node D experienced a LAN disconnect and a root account lockout, necessitating physical console intervention and manual IP assignment.

3. Recovery Logic

The mesh was stabilized using a Nix-First Recovery Protocol:

  1. Physical Rescue: Manually assigning 10.0.0.13 and restoring resolv.conf on Node D to allow Nix dependency fetching.
  2. Identity Alignment: Codifying the maczz user and PasswordAuthentication in common.nix to bridge the logical/physical gap.
  3. Remote Build-Host: Utilizing --build-host on the target nodes to bypass GPG signature requirements for unsigned closures.
  4. Artery Finalization: Sequential Tailscale authentication via a robust orchestration script (v3.1) that handles TTY allocation for sudo.

5. Lessons Learned

Never attempt to "sterilize" an active user identity without first establishing a backdoor Sovereign Key. Always map physical subvolumes in Nix before attempting a remote switch.

SSH Setup Required

Issue: Nodes A, C, D not reachable

nixos@10.0.0.10: Permission denied (publickey,password,keyboard-interactive).
nixos@10.0.0.12: Permission denied (publickey,password,keyboard-interactive).
nixos@10.0.0.13: Permission denied (publickey,password,keyboard-interactive).

Solution: SSH Key Configuration

Option 1: Setup SSH Keys on Node B

# Generate SSH key (if not exists)
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -N ""

# Copy public key to all nodes
ssh-copy-id nixos@10.0.0.10  # Node A
ssh-copy-id nixos@10.0.0.12  # Node C
ssh-copy-id nixos@10.0.0.13  # Node D

# Test connectivity
ssh nixos@10.0.0.10 "echo 'Node A reachable'"
ssh nixos@10.0.0.12 "echo 'Node C reachable'"
ssh nixos@10.0.0.13 "echo 'Node D reachable'"

Option 2: Manual Copy of SSH Keys

# Copy Node B SSH public key to each node manually
cat ~/.ssh/id_ed25519.pub | ssh nixos@10.0.0.10 "cat >> ~/.ssh/authorized_keys"
cat ~/.ssh/id_ed25519.pub | ssh nixos@10.0.0.12 "cat >> ~/.ssh/authorized_keys"
cat ~/.ssh/id_ed25519.pub | ssh nixos@10.0.0.13 "cat >> ~/.ssh/authorized_keys"

Option 3: Use Password Authentication Update deployment script to use password (not recommended):

# In deploy-phase2.sh, replace SSH commands with:
sshpass -p "password" ssh nixos@10.0.0.10 "command"

Troubleshooting

SSH Connection Failed

Symptom: Permission denied (publickey,password,keyboard-interactive)

Fix:

# Setup SSH keys
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519 -N ""
ssh-copy-id nixos@<node-ip>

Rust Build Failed

Symptom: error: failed to compile

Fix:

# Update Rust toolchain
rustup update

# Check dependencies
cargo tree

# Force rebuild
cargo clean && cargo build --release

Hermes Plugin Not Found

Symptom: Error: Plugin hermes-lcm not found

Fix:

# Check plugin path
ls -la ~/.hermes/plugins/

# Reinstall Hermes
cd ~/50V3R31GN-M4CH1N4/sidecars/hermes-agent-nous
pip install -e .

# Verify
hermes plugin list

Service Not Starting

Symptom: systemd: Service failed to start

Fix:

# Check logs
journalctl -u <service-name> -f

# Restart
systemctl restart <service-name>

# Check status
systemctl status <service-name>

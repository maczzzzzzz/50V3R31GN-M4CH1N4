Physical Node Sterilization Protocol (Beta v3)

Objective: The Quaternary Mesh (Nodes A, B, C, D) must be physically wiped of v2.1.0 prototype logic, stale processes, and orphaned containers before applying the Beta v3 Nix configurations.

THE IMPERATIVE

Our beta/v3 git repository is sterile, but if we deploy a clean Nix flake onto a dirty NixOS/Ubuntu host that has lingering systemd services, orphaned Docker containers, or manual .bashrc hacks from the prototype era, we will suffer from ghost collisions and port conflicts.

The physical layer must match the logical layer.

STERILIZATION STEPS (Execute per Node)

1. Identify & Preserve Resident Models (DO NOT NUKE)

Models (GGUFs, safetensors) take hours to download and consume massive disk space. Do not delete the models. - BACKUP PATH: The models are safely backed up on the Windows host at D:\llama.cpp\models. - NODE PATHS: Each node will have its own local copy of the required models. Identify where these local copies are stored on the node (e.g., /var/lib/ollama/models, ~/.cache/huggingface, or custom models/ directories) and ensure these directories are untouched during the purge.

2. Terminate Orphaned Daemons & Services

Kill any lingering prototype services that might occupy ports (8000, 8080, 9220, etc.).

# Check for rogue Python/Rust processes running prototype code
sudo systemctl stop hermes-agent sovereign-mcp warp-observer 2>/dev/null
sudo systemctl disable hermes-agent sovereign-mcp warp-observer 2>/dev/null

# Find and kill rogue node/python/cargo processes
pkill -f "sovereign"
pkill -f "browser-agent-harness"

3. Purge Stale Docker/Podman Containers

The prototype used several dockerized databases and bridges. Wipe them.

# WARNING: Ensure no critical non-project containers are running before executing
docker stop $(docker ps -aq --filter "name=sovereign|hermes|mempalace|mooncake") 2>/dev/null
docker rm $(docker ps -aq --filter "name=sovereign|hermes|mempalace|mooncake") 2>/dev/null

4. Nix Store Garbage Collection (Optional but Recommended)

If the nodes run NixOS, clear out the old derivations to reclaim disk space.

# Run this on NixOS nodes to purge dead code paths
nix-collect-garbage -d

5. Verify Network Sterility

Before deploying Beta v3, verify that the critical ports are silent.

# These ports MUST return nothing (meaning they are free)
sudo lsof -i :8000 # Hermes API
sudo lsof -i :8080 # IK Llama.cpp
sudo lsof -i :9119 # Pretext / Hermes Desktop
sudo lsof -i :7878 # VSB UDP / MCP Bridge

THE TRANSITION TO IMMUTABILITY

Once this sterilization is complete, the physical nodes are “dumb metal.” From this point forward, no manual configuration is allowed on the nodes.

GLM-5 will build the flake.nix in Phase 1 and Phase 2. We will use nixos-rebuild switch --flake .#node-[x] to push the configuration. The Nix flake will be the absolute and only source of truth for what software runs on the physical metal.

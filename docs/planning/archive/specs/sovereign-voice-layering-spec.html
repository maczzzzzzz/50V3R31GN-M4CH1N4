◈ SOVEREIGN VOICE LAYERING SPECIFICATION (V3-BETA)

Status: DRAFT | Target Phase: 2 (Pluggable Sovereign Layer)

1. STRATEGIC OBJECTIVE

To architect a zero-trust, high-fidelity voice pipeline that wraps the Hermes native agentic core with the Sovereign amplification stack. This ensures studio-quality perception and response while maintaining physical hardware independence (Omi/PC parity).

2. THE QUATERNARY VOICE TOPOLOGY

Component Node Technology Role
Capture B / Client Omi SDK / WASAPI / PipeWire Multi-source physical input.
Duplex Core C Moshi-RAG Full-duplex speech + asynchronous RAG grounding.
STT Base D Hermes Native (Whisper) Initial transcription hook (Fallback).
ASR Amp D VibeVoice (Microsoft) Diarization, structured metadata, long-form coherence.
Brain D Qwen3-14B-V2-27B (Hermes Core) Agentic reasoning and tool dispatch.
TTS Amp D VoxCPM2 (OpenBMB) 48kHz Expressive synthesis / Zero-shot cloning.
Perception C Falcon Perception stack Real-time threat/environment awareness.

3. DATA FLOW (THE COGNITIVE ARTERY)

  1. Input: Raw PCM audio streamed from Omi (BLE/WiFi) or PC Headset via Clawlink Bridge to Node D.
  2. Transcribe:
    • Hermes Whisper generates raw text.
    • VibeVoice-ASR processes the same stream to provide speaker ID and structured context (Timestamped).
  3. Process: Hermes Core receives the enriched transcript, logs it to MemPalace, and generates a response.
  4. Synthesize:
    • VoxCPM2 generates the response audio using the Ultimate Cloning path (referencing the user’s Omi audio for zero-latency identity matching).
    • Output is 48kHz studio-quality stream.
  5. Return: Audio streamed back to the source device (Flutter App or PC Speakers).

4. IMPLEMENTATION MODULES (PHASE 2)

Module V1: crates/modules/vibevoice-asr

  • Engine: Python bridge via PyTorch/vLLM.
  • Task: Wrap the 7B model for diarized transcription.
  • Nix: FHS container for CUDA 12.x dependencies.

Module V2: crates/modules/voxcpm-tts

  • Engine: Python bridge via vLLM-Omni.
  • Task: 48kHz synthesis with voice design capability.
  • Nix: FHS container for 48kHz audio drivers and MiniCPM backbone.

Module V3: sidecars/omi-sdk-bridge

  • Engine: Python SDK / MCP.
  • Task: Connect Omi app webhooks to the Node D pipeline.

5. REASONING INVARIANTS

  • Zero-Latency Percept: Transcription must occur within <300ms.
  • Sovereign Audit: Every voice episode is audited by Psy-core before TTS return.
  • Privacy: Local processing only. No cloud STT/TTS permitted in high-security mode.

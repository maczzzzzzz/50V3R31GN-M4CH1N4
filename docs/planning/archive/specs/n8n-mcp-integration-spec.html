SPEC_2026-05-05: n8n-mcp Integration (Phase 4)

Objective: Externalize complex API orchestration from the Hermes Core into an auditable, visual workflow layer to eliminate “Shadow Logic.”

1. Strategic Rationale

  • Logic Externalization: Moves integrations (Discord, Telegram, Home Assistant) into n8n workflows.
  • Auditability: Visual canvas allows for zero-trust inspection of agent “intent.”
  • Efficiency: Rapidly prototype tool artifacts without Nix flake rebuilds.

2. Technical Architecture

  • Hosting: Self-hosted n8n instance on Node B (or dedicated container).
  • Bridge: Deploy n8n-mcp server on Node D to expose workflows as native Hermes tools.
  • Nix-ification: Both n8n and the MCP bridge must be defined as reproducible Nix modules in flake.nix.

3. Governance

  • No n8n Cloud allowed.
  • All workflows must be version-controlled as JSON within the docs/nodestadt/workflows/ directory.

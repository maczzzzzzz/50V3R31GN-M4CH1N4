Node Bootstrap Procedure

Process for bringing a new physical node into the mesh.

Prerequisites

  • Physical machine with NixOS installed (or WSL2 for Node B)
  • Network connectivity (Ethernet or WiFi)
  • Tailscale account access for tailnet

Step 1: Base NixOS Install

Install NixOS 24.11 (Vicuna) with standard partition layout. The mesh uses Btrfs with @ and @home subvolumes, systemd-boot, and a swap partition.

Step 2: Tailscale

# Install and authenticate
nix-env -iA nixos.tailscale
sudo tailscale up

# Verify connectivity
tailscale status

Step 3: SSH Keys

# From Node B (workspace), deploy key to new node
ssh-copy-id nixos@<tailscale-ip>

# Verify headless access
ssh nixos@<tailscale-ip> "hostname"

Ensure Tailscale ACL includes the SSH policy. Without it, connections will prompt for browser-based authentication.

Step 4: Flake Config

Clone the mesh repo and apply node-specific config:

git clone https://github.com/maczzgit/50V3R31GN-M4CH1N4.git
cd 50V3R31GN-M4CH1N4-stable-mesh-alpha

# Dry-run first
nix build .#nixosConfigurations.node-c.config.system.build.toplevel

# Apply
sudo nixos-rebuild switch --flake .#node-c

Step 5: GPU Driver

  • NVIDIA nodes (A, C): Verify hardware.nvidia is in the config, reboot, check nvidia-smi
  • AMD nodes (B): Vulkan is used, not ROCm. Verify via vulkaninfo
  • CPU nodes (D): No driver needed. Verify AVX2 support: lscpu | grep avx2

Step 6: Inference Runtime

Build llama.cpp or ik_llama.cpp for the appropriate backend and deploy the assigned model. See each node's detail page for specifics. Node B uses ik_llama.cpp Vulkan, Node C uses ik_llama.cpp CUDA, Node D uses stock llama.cpp AVX2.

Step 7: Register with LiteLLM

Add the node's inference endpoint to the LiteLLM config on Node B (port 4000). See the model strategy page for the routing matrix.

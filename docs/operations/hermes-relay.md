# Hermes-Relay Service (v0.6.1)

The **hermes-relay** is a Phase 1 bridge service providing WebSocket (WSS) and HTTP connectivity for mobile (Phone) and terminal interfaces. It acts as the primary gateway for the Hermes Bridge and Terminal channels.

---

## 1. Specifications

- **Version:** v0.6.1
- **Deployment:** Docker container (`hermes-relay`)
- **Port:** `8767`
- **Network Mode:** Docker bridge with port mapping (0.0.0.0:8767 -> 8767/tcp)
- **Restart Policy:** `unless-stopped`
- **Health Check:** `http://localhost:8767/health`

---

## 2. Architecture

The relay sits between the Hermes API and external clients (typically the Hermes Android client).

- **Phone -> WSS :8767**: Terminal and Bridge channels for real-time interaction.
- **Phone -> HTTP :8642**: Direct chat and tool execution (via Hermes API Gateway).
- **Relay -> Hermes API**: Uses `host.docker.internal:8642` to communicate with the local Hermes instance.

---

## 3. Configuration

The service is configured via environment variables and volume mounts:
- **Mounts:** `~/.hermes` is mounted **read-only** to provide access to shared configuration and identity.
- **API URL:** `HERMES_API_URL=http://host.docker.internal:8642`

### Docker Run Command (Reference)
```bash
docker run -d \
  --name hermes-relay \
  -p 8767:8767 \
  --restart unless-stopped \
  --user 1000:100 \
  -e HOME=/home/nixos \
  -e RELAY_HOST=0.0.0.0 \
  -e RELAY_PORT=8767 \
  -e RELAY_WEBAPI_URL=http://host.docker.internal:8642 \
  -e RELAY_HERMES_CONFIG=/home/nixos/.hermes/config.yaml \
  -e RELAY_LOG_LEVEL=INFO \
  -v /home/nixos/.hermes:/home/nixos/.hermes:ro \
  hermes-relay:latest
```

Note: `--user 1000:100` matches the nixos UID so the container can read the mounted config. `host.docker.internal` is required because Docker Desktop WSL2 `--network host` does not bridge to WSL2 localhost.

---

## 4. Authentication

Access is secured via a pairing mechanism:
1. **Pairing Code:** 6-character alphanumeric code generated by the relay.
2. **Session Tokens:** 30-day session tokens issued upon successful pairing.
3. **Storage:** Sessions are persisted in the relay's internal state.

---

## 5. Hermes Plugin: `hermes-relay`

The relay functionality is extended by a Hermes plugin located at `~/.hermes/plugins/hermes-relay/`.

- **Tools:** 18 specialized `android_*` tools for phone interaction (e.g., `android_navigate`, `android_notifications`, `android_phone_status`).
- **Dashboard:** A built-in React-based management UI for relay status and pairing.
- **Relay Logic:** Python-based handlers for various channels (chat, bridge, tui, terminal).

---

## 6. Maintenance & Health

- **Check Status:** `docker ps | grep hermes-relay`
- **Check Health:** `curl http://localhost:8767/health`
- **Logs:** `docker logs -f hermes-relay`

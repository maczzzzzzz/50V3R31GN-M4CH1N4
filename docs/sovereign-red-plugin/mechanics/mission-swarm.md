## ⚡ THE MISSION SWARM
The **Mission Swarm** is a concurrent reasoning shard designed for the on-the-fly generation of high-fidelity, rules-accurate Cyberpunk RED operations. It fuses mechanical precision with deep lore to materialize a "Mission Blueprint" within the simulation.

## 1. THE SYNTHESIS SEQUENCE
1. **RULES_RECKONING (NODE_A)**: The **Open-Reasoner-1.5B** executes grounded calculations for Difficulty Values (DVs) and encounter tables mapped to the target district.
2. **TACTICAL_OVERVIEW (NODE_B)**: **Mistral-Nemo-12B** performs semantic analysis on the mission objective, providing a narrative-tactical blueprint.
3. **LORE_ANCHORING**: The engine queries **Akashik.db** for recent entity transcripts and events, ensuring the operation is tethered to the session's historical trajectory.

---

## 2. IGNITION PROTOCOLS
Operations are triggered via the simulation's command interface:
```bash
/mission [District]
```
*EXAMPLE*: `/mission Combat_Zone_Alpha`

---

## 3. BLUEPRINT DATA-PACKAGE
The materialized blueprint includes:
- **NARRATIVE_OBJECTIVE**: The primary mission goal and stakes.
- **MECHANICAL_INTEL**: Precision DVs for core tasking (Security, Netrunning, Stealth).
- **LORE_ANCHORS**: Dynamic integration of known NPCs and locations.
- **ATMOSPHERIC_CONFIG**: Specialized lighting and "glitch" CSS parameters for the Shroud interface.

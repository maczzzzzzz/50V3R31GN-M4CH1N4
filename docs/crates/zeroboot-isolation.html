zeroboot-isolation

KVM/Firecracker MicroVM Isolation with SCION Networking

Back to Crates Index

Overview

Zeroboot Isolation provides containerized agent workspaces using KVM-based Firecracker microVMs with SCION networking for multi-agent isolation. The design prioritizes fast spawn/teardown (target: <2s) and minimal overhead (512MB base image). Lifecycle management uses zero-fork /proc polling instead of spawning external processes for status checks.

Key Types

VMConfig

MicroVM configuration:

  • id: String — VM identifier
  • vcpus: u8 — CPU count (default: 2)
  • memory_mb: u64 — Memory allocation (default: 2048)
  • workspace_path: PathBuf — Workspace directory (uses XDG_RUNTIME_DIR)

Factory method: VMConfig::agent_default(id) provides sensible defaults for agent workspaces.

MicroVM

Firecracker microVM runtime:

  • new(config) — Create workspace directory
  • launch() — Spawn Firecracker process, detach, track PID
  • stop() — Send SIGTERM via libc::kill (zero-fork)
  • is_alive() — Check /proc/{pid}/stat exists (zero-fork)

SCIONIsolation

SCION networking wrapper around MicroVM:

  • new(config) — Create VM + resolve SCION binary path
  • launch() — Launch VM then apply SCION network isolation

Design Decisions

  • Zero-fork lifecycle — Uses libc::kill() syscall directly instead of spawning kill command. Status checks via /proc filesystem. No child process overhead.
  • Detached spawn — Firecracker is spawned and immediately detached (child handle dropped). Only the PID is retained for lifecycle management.
  • XDG_RUNTIME_DIR — Workspace paths default to XDG runtime directory for proper ephemeral storage.

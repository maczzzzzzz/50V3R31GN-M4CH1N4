vibevoice-asr

Speech Recognition Layer with Multi-Source Audio

Back to Crates Index

Overview

VibeVoice ASR is a multi-source speech recognition pipeline built around Whisper. It supports three audio sources with automatic priority resolution: Omi hardware BLE streaming (highest priority), mobile device microphone via Tailscale, and local file input. The VibeVoice post-processor adds emotion/vibe scoring to transcriptions for agent context enrichment.

The BLE feature flag (--features ble) enables the full Omi hardware integration via btleplug. Without it, the crate operates in file-only mode.

ASRRequest

  • audio_file: String — Path to audio file
  • language: String — Language code (default: "en")
  • post_process: bool — Enable VibeVoice scoring (default: true)
  • source: AudioSource — Audio source (auto-resolved)

ASRResponse

  • text: String — Transcribed text
  • confidence: f64 — Confidence score (0.0-1.0)
  • vibe_score: Option<f64> — Emotion/style score
  • source: AudioSource — Source that provided the audio

BleStreamManager (feature = "ble")

BLE device lifecycle: adapter init, Omi device scan (Sovereign Artery UUID), connection, PCM notification subscription, buffered sample reading, clean disconnect.

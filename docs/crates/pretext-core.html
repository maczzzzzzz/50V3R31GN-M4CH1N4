pretext-core

Unified Text Engine for Zero-DOM Layout

Back to Crates Index

Overview

Pretext Core provides bit-identical layout arithmetic across two rendering targets: Node B's React/WASM frontend and the Machina Terminal's Flutter/FFI interface. All layout math is pure Rust geometric calculations — no DOM dependency, no browser-specific behavior. The engine also includes an ASCII brightness mapper for terminal rendering and a thought stream virtualizer for HUD display.

The dual crate-type build (cdylib + rlib) ensures the same code compiles to both a WASM module and a native C library from a single source.

Key Types

PretextEngine

Core layout engine. Takes font family, font size, and text input. Produces LayoutResult with lines, segments, widths, and heights — all as pure float arithmetic, no DOM calls.

LayoutResult / LayoutLine / TextSegment

Structured layout output:

  • LayoutResult — tight_width, total_height, lines
  • LayoutLine — width, segments
  • TextSegment — text, width, char_count

ASCII Mapper

Public functions for terminal rendering:

  • brightness_to_ascii(f32) -> char
  • text_to_ascii_brightness(&str, f32) -> String
  • token_ascii_mapping(&str) -> String
  • gradient_ascii_typography, pulse_ascii_typography, wave_ascii_typography, variable_ascii_typography

ThoughtStreamVirtualizer

HUD display virtualization for thought nodes. Produces VirtualLayoutResult with VirtualThoughtNode entries for rendering KineticThoughtStream in the dashboard.

FFI Bindings (Flutter)

Full C FFI surface for Flutter integration:

  • pretext_engine_create / pretext_engine_destroy — Engine lifecycle
  • pretext_layout / pretext_layout_destroy — Layout computation
  • pretext_layout_get_lines, get_lines_count, get_tight_width, get_total_height
  • pretext_layout_line_get_segments, get_segments_count, get_width
  • pretext_layout_segment_get_text, get_text_len, get_width, get_char_count
  • ffi_ascii_brightness_to_char, ffi_ascii_text_brightness

All FFI functions include null pointer checks and UTF-8 validation via the internal ffi_str helper.

mirage-vfs

Virtualized Filesystem Client

Back to Crates Index

Overview

Mirage VFS presents external data sources (Redis on Node A, S3) as local files via a FUSE mount on Node D. It provides a clean Rust API for mount/unmount lifecycle, health checks, and transparent file read/write operations through the virtual filesystem. This is the best-engineered crate in the mesh — well-structured modules, comprehensive error types, and thorough path traversal protection.

Key Types

MirageVfsError

Structured error enum with thiserror derives:

  • MountPoint(String) — Mount point access failure
  • MountFailed(String) — FUSE mount operation failed
  • UnmountFailed(String) — FUSE unmount failed
  • HealthCheckFailed(String) — Mount not active
  • IoError(io::Error) — File I/O on virtual filesystem
  • ConfigError(String) — Configuration error

MountStatus

Enum representing mount state: Mounted, Unmounted, NotFound.

HealthReport

Full health check result including:

  • status: MountStatus
  • mount_point: PathBuf
  • redis_reachable: bool
  • s3_reachable: bool
  • entries: Vec<String> — Top-level VFS listing
  • timestamp: u64 — Epoch seconds

MirageVfs (Client)

Main client with full lifecycle:

  • new(config) / from_env() — Initialize from config or env vars
  • mount() / unmount() — FUSE lifecycle
  • mount_status() / health_check() — Status inspection
  • read_file(path) / write_file(path, data) — Transparent VFS I/O
  • list_dir(path) — Directory listing

Security

  • Dual path traversal protection — Validates resolved path stays within mount point AND tracks .. component depth
  • Canonical path resolution — Handles both existing and new-file paths correctly
